# ADR-005: Central styles/ Directory with Domain CSS Files

## Status

Accepted

## Date

2026-06-21

## Context

`simpa-frontend/src/index.css` is ~2900 lines mixing tokens, layout, painel, cadastros, admin, importacao. Tailwind 4 uses `@import 'tailwindcss'` at root.

## Decision

Split into `simpa-frontend/src/styles/`:

| File | Contents |
|------|----------|
| `tokens.css` | `:root`, `[data-theme='dark']` variables |
| `base.css` | reset, body, `.card`, `.mono` |
| `layout.css` | sidebar, topbar, shell, filter bar |
| `painel.css` | painel layouts, KPI, charts container |
| `cadastros.css` | cadastro CRUD, tables, dialogs |
| `admin.css` | admin subnav, backup, config forms |
| `importacao.css` | upload zone, mapping panels |

`index.css` becomes:

```css
@import 'tailwindcss';
@import './styles/tokens.css';
@import './styles/base.css';
/* ... */
```

**No visual redesign** — move rules only; verify dark theme parity.

## Alternatives Considered

### Co-locate CSS modules per component

- **Pros:** True colocation.
- **Cons:** Large migration; many components use global class names today.
- **Why rejected:** Domain files match current global CSS pattern with less churn.

### Leave monolithic index.css

- **Pros:** Zero CSS risk.
- **Cons:** Fails PRD maintainability goal.
- **Why rejected:** Phase 1 deliverable.

## Consequences

### Positive

- Developers edit `admin.css` for Backup page without scanning painel rules.

### Negative

- One-time risk of missing a rule during move — mitigate with visual smoke + Playwright.
