# ADR-003: Gate de reimportação por competência (s_prd)

**Status:** Accepted  
**Date:** 2026-06-21  
**Context:** `sia_sincronizacoes` tem `UNIQUE(competencia)` e `gravar_pg` já faz DELETE+INSERT na mesma sync — tecnicamente idempotente, mas **silencioso**: operador não sabe que vai sobrescrever; risco de clique duplo ou importação acidental.

## Decision

1. **Verificar antes de importar:** consultar `sia_sincronizacoes` para `competencia` com `status IN ('ok','parcial')`.
2. **Se já importada e `reimportar !== true`:** API retorna **409 Conflict** com corpo `{ code: 'SIA_COMPETENCIA_JA_IMPORTADA', competencia, sincronizado_em, registros }` — UI exibe aviso.
3. **Se operador confirma reimportação:** `POST { competencia, reimportar: true }` → ETL executa:
   - `DELETE FROM sia_producao WHERE competencia = $1` (por data, não só sincronizacao_id)
   - Re-extrai MySQL e grava agregado
   - Atualiza linha em `sia_sincronizacoes`
4. **Primeira importação:** `reimportar` omitido ou `false` → fluxo normal.

## UI

- `ConfirmDialog` (padrão SIMPA): “Competência YYYY-MM já importada em DD/MM/AAAA (N registros). Deseja reimportar? Os dados atuais serão substituídos.”
- Botões: Cancelar / Reimportar

## Cadastros (forma, CBO, procedimento, rubrica)

Sync de **cadastros** permanece **snapshot UPSERT** (sem competência) — reexecutar `POST /api/cadastros/sincronizar` já atualiza forma/cbo/procedimento sem duplicar (ON CONFLICT). Rubricas (`s_rub`) passam a espelho `rubricas_sia` no mesmo script (ADR complementar na task_08).

**Ordem recomendada na UI produção:** oferecer link “Atualizar cadastros SIA” antes da primeira importação de produção da competência.

## Alternatives considered

| Opção | Prós | Contras |
|-------|------|---------|
| Sempre sobrescrever silencioso (atual) | Simples | UX ruim; sem auditoria de intenção |
| Bloquear reimport permanente | Zero duplicação | Impossível corrigir sync |
| **409 + confirm (escolhida)** | Explícito; DELETE garantido | Um passo a mais na UI |

## Consequences

- Endpoint opcional `GET /api/sia/sincronizacoes/existe?competencia=YYYY-MM` para check sem side-effect.
- Testes: 409 sem reimportar; 201 com reimportar após DELETE.
