# ADR-004: Espelho rubricas_sia a partir de s_rub

**Status:** Accepted  
**Date:** 2026-06-21  
**Context:** Produção SIA referencia `PRD_RUB` → `s_rub.RUB_ID`. Procedimentos trazem `RUB_TOTAL` no cadastro SIGTAP, mas a **dimensão de financiamento** (`s_rub`: MAC, AB, FAEC…) não está espelhada em PG. Enriquecimento de produção e relatórios precisam de `RUB_DC`.

## Decision

1. Criar tabela `rubricas_sia` (PK `codigo_rubrica` char(4), `descricao`, `status`, `sincronizado_em`).
2. Estender `sync_cadastros_mysql.py` com `extrair_rubricas` / `sync_rubricas` de MySQL `s_rub`.
3. Incluir contadores `rubricas_{inserted,updated,inactivated}` em `cadastros_sincronizacoes` (migration 011 ou colunas JSON em payload — preferir colunas espelhando forma/cbo).
4. Reimport cadastros = UPSERT idempotente (mesmo padrão forma/cbo) — **sem gate por competência**.

## Consequences

- `GET /api/cadastros/rubricas` read-only (Phase 2 UI opcional).
- Join produção: `dados_extras.rubrica_codigo` → `rubricas_sia.descricao`.
