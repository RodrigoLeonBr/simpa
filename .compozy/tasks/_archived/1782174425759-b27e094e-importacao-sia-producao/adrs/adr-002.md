# ADR-002: Seleção de campos s_prd e grão de agregação

**Status:** Accepted  
**Date:** 2026-06-21  
**Context:** `s_prd` expõe dezenas de colunas administrativas (folha, sequência, flags de erro, APAC, CNS, NF, CIDs secundários) irrelevantes para Painel MAC, relatórios gerenciais e API `/sia/producao`. Importar linha bruta infla PostgreSQL sem valor.

## Decision

1. **Importar apenas competência escolhida** (`prd_cmp = AAAAMM`) — operador seleciona ano/mês na UI ou CLI.
2. **Totalizar no MySQL** (GROUP BY + SUM) antes de gravar em PG — nunca persistir `prd_flh`, `prd_seq` nem demais colunas excluídas.
3. **Grão agregado fixo:**

```
competencia × cnes × codigo_sigtap × faixa_etaria × sexo × cbo × rubrica_codigo
```

4. **Métricas somadas:** `PRD_QT_P`, `PRD_QT_A`, `PRD_VL_P`, `PRD_VL_A` (com CAST).
5. **Dimensões descritivas:** nome unidade (`re_cnome`), descrição procedimento — via JOIN, não duplicadas por folha.

### Campos s_prd excluídos (nunca SELECT nem PG)

| Coluna | Motivo |
|--------|--------|
| `prd_flh`, `prd_seq` | Chave operacional folha — eliminados pela agregação |
| `PRD_ORG` | Órgão emissor — não usado em KPIs SIMPA |
| `PRD_FLPA` … `PRD_FLER` | Flags de erro — auditoria billing, fora do escopo |
| `PRD_APANUM` | APAC — escopo hospitalar/APAC futuro |
| `PRD_CNSMED` | CNS profissional — não agregamos por profissional |
| `PRD_CNPJ` | Já em `prestador`/cadastro |
| `PRD_NFIS` | Nota fiscal |
| `PRD_CIDSEC`, `PRD_CIDCAS` | CID secundário/causa — fora do MVP |

### Campos s_prd incluídos

| Coluna | Destino PG |
|--------|------------|
| `prd_cmp` | filtro → `competencia` |
| `prd_uid` | `cnes` + FK `estabelecimento_id` |
| `prd_pa` | `codigo_sigtap` |
| `prd_cbo` | `cbo` |
| `PRD_IDADE` | derivado → `faixa_etaria` |
| `PRD_QT_P` / `PRD_QT_A` | `quantidade_apresentada` / `quantidade` |
| `PRD_VL_P` / `PRD_VL_A` | `valor_apresentado` / `valor_aprovado` |
| `PRD_RUB` | `dados_extras.rubrica_codigo` (+ descrição via `s_rub`) |

### Campos adiados (Phase 2+)

`PRD_MVM`, `PRD_RMS`, `PRD_CPX`, `PRD_TPFIN`, `PRD_CIDPRI` — só se relatório clínico exigir.

## Alternatives considered

| Opção | Prós | Contras |
|-------|------|---------|
| Espelho 1:1 com todas colunas | Fidelidade máxima | PG gigante; folha/seq irrelevantes |
| Agregar só QT/VL por competência total | Mínimo volume | Perde drill-down procedimento/unidade |
| **Agregar no grão gerencial (escolhida)** | Balance volume × utilidade | Perde folha/seq/flags |

## Consequences

- Redução estimada: de N linhas raw (folha×seq por competência) para ~1 linha por combinação de dimensões gerenciais (tipicamente 2–4 ordens de magnitude menor).
- Re-sync idempotente: DELETE por competência + INSERT agregado.
- Qualquer necessidade futura de flags/APAC exige ADR novo e possivelmente tabela separada.
