# ADR-001: Sync por competência com agregação no ETL

**Status:** Accepted  
**Date:** 2026-06-21  
**Context:** `s_prd` tem ~5.9M registros granulares (folha/sequência). SIMPA não precisa replicar linha bruta do MySQL — Painel e API agregam por procedimento × faixa × sexo × CBO × unidade.

## Decision

Extrair do MySQL com `GROUP BY` no grão gerencial (ADR-002), filtrando **sempre** `prd_cmp` escolhido pelo operador. Colunas administrativas (folha, sequência, flags) são eliminadas pela agregação — não SELECT, não PG.

Ver também [ADR-002](adr-002.md) para lista completa de campos excluídos.

## Alternatives considered

| Opção | Prós | Contras |
|-------|------|---------|
| Espelho raw 1:1 | Máxima fidelidade | Volume PG enorme; sync lento |
| Query on-demand MySQL | Sem cópia | Timeout; acoplamento runtime |
| **Agregado por competência (escolhida)** | Alinha consolidador; volume controlável | Perde folha/sequência (não usada hoje) |

## Consequences

- UNIQUE key em `sia_producao` permanece no grão agregado.
- Relatórios futuros que precisem folha/sequência exigem ADR novo.
