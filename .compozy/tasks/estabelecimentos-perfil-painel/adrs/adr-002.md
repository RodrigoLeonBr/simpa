# ADR-002: Preserve Manual Perfil via `perfil_editado` Flag

## Status

Accepted

## Date

2026-06-20

## Context

MySQL cadastro sync upserts `estabelecimentos` and currently sets `perfil = EXCLUDED.perfil` on every conflict, overwriting any manual correction. PRD requires preserving user-edited Perfil while still deriving Perfil on first insert from `tipouni`.

## Decision

Add column `perfil_editado BOOLEAN NOT NULL DEFAULT false` to `estabelecimentos`.

- **INSERT (new row):** set `perfil` from `derive_perfil(tipouni)`, `perfil_editado = false`.
- **UPDATE on sync conflict:** update identity fields (`nome`, `cnpj`, `re_tipo`, `tipouni`, `area`, `relatorio`, `status`, `sincronizado_em`) always; update `perfil` **only when** `perfil_editado = false`.
- **PUT `/estabelecimentos/:id/perfil`:** set `perfil` to allowed value and `perfil_editado = true`.

Expose `perfil_editado` in GET responses (read-only) for UI hinting optional in Phase 2.

## Alternatives Considered

### Alternative 1: Omit `perfil` from UPSERT UPDATE entirely

- **Pros:** No new column.
- **Cons:** If `tipouni` changes in MySQL, derived perfil never updates even when user never edited; no signal of manual override.
- **Why rejected:** Cannot distinguish "never edited" from "edited once then tipouni changed".

### Alternative 2: Store `perfil_derivado` alongside `perfil`

- **Pros:** UI can show drift vs SIA.
- **Cons:** Extra column and sync logic for marginal MVP value.
- **Why rejected:** YAGNI for Phase 1; `tipouni` remains visible read-only.

## Consequences

### Positive

- Explicit, testable sync rule.
- New establishments still get auto-classification.
- Manual corrections survive re-sync.

### Negative

- Requires migration `005` and Python UPSERT change.
- Users cannot "reset to derived" without new UI action (defer to Phase 2).

### Risks

- **Risk:** Stale manual perfil after municipal reclassification — **Mitigation:** show `tipouni` read-only; future "revert to derived" action.

## Implementation Notes

- Change `UPSERT_ESTABELECIMENTO_SQL` in `sync_cadastros_mysql.py` to conditional `perfil` update.
- `updatePerfil` in `estabelecimentosService.js` validates against `VALID_PERFIS`.
