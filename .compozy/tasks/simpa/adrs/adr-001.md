# ADR-001: Docker Compose Single-Server Deployment

## Status

Accepted

## Date

2026-06-19

## Context

SIMPA must run entirely on one municipal server (PostgreSQL, Node API, React frontend, Python ETL). The Secretaria may use Windows with XAMPP for legacy SIA MySQL. Reproducible deployment and clear service boundaries are required for a small/medium team without dedicated DevOps.

## Decision

Deploy SIMPA using **Docker Compose** with four services:

1. **postgres** — PostgreSQL 15, persistent volume, init from `schema_full.sql` + migrations
2. **api** — Node.js 18 Express; includes Python 3.11 runtime for ETL subprocesses; mounts `uploads/` and project Python scripts
3. **web** — Nginx serving Vite production build; reverse-proxies `/api/*` and `/auth/*` to `api:3001`
4. **No MySQL container** — SIA reads from host XAMPP MySQL via `host.docker.internal`

Host ports: `80` (web), `5432` optional expose for admin only.

## Alternatives Considered

### Alternative 1: Native PM2 + Apache (XAMPP)

- **Description**: PostgreSQL native, Node via PM2, React dist in Apache htdocs
- **Pros**: Familiar to XAMPP users, no Docker overhead
- **Cons**: Manual dependency management, harder reproducibility across environments
- **Why rejected**: User chose Docker Compose for operational consistency

### Alternative 2: Full stack in Docker including MySQL

- **Description**: Import SIA dump into a MySQL container
- **Pros**: Fully containerized, no host coupling
- **Cons**: Duplicate of live XAMPP data; sync burden
- **Why rejected**: User chose XAMPP host MySQL as SIA source of truth

## Consequences

### Positive

- One `docker compose up` brings up the full application stack
- Nginx handles TLS termination and static assets in production
- ETL scripts share the API container filesystem with `uploads/`

### Negative

- Docker Desktop required on Windows host
- `host.docker.internal` adds platform-specific networking configuration

### Risks

- **Risk**: XAMPP MySQL unreachable from container — **Mitigation**: document firewall rules and `extra_hosts` in compose
- **Risk**: Large CSV uploads — **Mitigation**: configure nginx `client_max_body_size` and multer limits

## Implementation Notes

- Multi-stage `Dockerfile.frontend` builds React to `dist/`
- API `Dockerfile` installs Python deps from `requirements.txt`
- `.env` at repo root consumed by compose; secrets not committed
- Development override `docker-compose.dev.yml` exposes Vite HMR on 5173
