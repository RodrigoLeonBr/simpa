# ADR-004: Basic JWT Auth Without API-Level RBAC in MVP

## Status

Accepted

## Date

2026-06-19

## Context

LGPD requires profile-based access (Administrador, Gestor Secretaria, Gestor de Unidade, Planejamento). Full API-level scoping (auto-filter by unit for Gestor de Unidade) requires `usuarios` linked to units, middleware on every endpoint, and audit trails. The user chose basic auth for MVP to reduce delivery time while keeping the login UX from `SIMPA.dc.html`.

## Decision

Implement **JWT authentication** with:

- Table `usuarios` (bcrypt password hash, perfil enum)
- Routes: `POST /auth/login`, `POST /auth/logout`, `GET /auth/me`
- Middleware `verifyJWT` on all `/api/*` routes
- Frontend: `AuthContext`, `ProtectedRoute`, login page matching prototype
- **No automatic data scoping by perfil in API** — users filter via FilterBar manually
- Profile displayed in topbar; perfil select on login is **informational only** (server validates username/password; perfil comes from DB record)

Defer to Phase 2: unit/equipe FK on `usuarios`, API filter injection, full `audit_log` on exports.

## Alternatives Considered

### Alternative 1: Full RBAC from MVP

- **Description**: API enforces unit scope per perfil
- **Pros**: LGPD-compliant from day one
- **Cons**: Requires user-unit mapping, migration, extensive middleware tests
- **Why rejected**: User chose basic auth for faster MVP

### Alternative 2: No auth in MVP

- **Description**: Open dashboard behind VPN only
- **Pros**: Fastest
- **Cons**: Unacceptable for health data even if aggregated
- **Why rejected**: Auth design spec and prototype include login

## Consequences

### Positive

- Login flow and JWT infrastructure ready for Phase 2 RBAC extension
- Simpler API queries without scope logic

### Negative

- Gestor de Unidade could query other units if they change FilterBar — mitigated by organizational policy until Phase 2
- Login perfil dropdown may confuse if it doesn't match DB perfil — frontend should hide dropdown or sync from DB response only

### Risks

- **Risk**: LGPD audit gap — **Mitigation**: log login events in `audit_log` minimally; document Phase 2 scope enforcement

## Implementation Notes

- Remove perfil from login POST body; return perfil from server only
- Seed admin user: `admin` / `simpa@2026` (change on first deploy)
- `JWT_EXPIRES_IN=8h`
