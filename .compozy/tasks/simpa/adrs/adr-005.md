# ADR-005: React Port of SIMPA.dc.html Full Redesign

## Status

Accepted

## Date

2026-06-19

## Context

`SIMPA_ tela/SIMPA.dc.html` is the validated UI prototype with login, app shell (236px sidebar), Painel layouts A/B/C, Indicadores, Metas, Importação, Relatórios, Cadastros grid, Administração, and Sala de Situação telão. The dc-runtime HTML is a design reference, not a production runtime. User selected **full redesign** scope for MVP.

## Decision

Build **React + Vite + TypeScript** frontend that faithfully ports the prototype visual and navigation:

- Design tokens from `docs/design-system.md` (IBM Plex Sans/Mono, CSS custom properties, light default + dark mode)
- 7 menu modules + login + Sala de Situação overlay (no route change)
- Painel: layouts A/B/C switcher (replaces old APS/MAC/Hospitalar tabs from Plano C)
- All CRUD cadastro sub-routes: Unidades, Equipes, Procedimentos, Prestadores MAC, Hospitais, Emendas
- Admin: Usuários, Audit Log (read-only), Configurações
- Export Excel/PDF buttons present with toast "Em breve"
- ECharts for all charts; map placeholder SVG in Relatórios

Do **not** ship dc-runtime in production; use it only as visual reference.

## Alternatives Considered

### Alternative 1: Embed dc-runtime directly

- **Description**: Serve SIMPA.dc.html with minimal wrapper
- **Pros**: Fastest to pixel-perfect prototype
- **Cons**: No type safety, poor testability, no real API integration pattern
- **Why rejected**: Production needs React ecosystem, auth, and ECharts integration

### Alternative 2: MVP Core only (Painel + Importação)

- **Description**: Placeholder pages for other modules
- **Pros**: Smaller scope
- **Cons**: Diverges from user-approved full redesign
- **Why rejected**: User selected full redesign scope

## Consequences

### Positive

- Single design system document governs all screens
- Prototype screens map 1:1 to React routes/components for task breakdown

### Negative

- Largest frontend surface area; requires disciplined component reuse (FilterBar, KPI cards, progress bars)

### Risks

- **Risk**: Visual drift from prototype — **Mitigation**: design-system.md tokens; screenshot comparison in QA checklist

## Implementation Notes

- Component folder structure per auth design spec Section 14
- `indicadores_qualidade[]` added to dashboard contract TypeScript types
