# ADR-006: Full Test Strategy (Unit + API Integration + Playwright E2E)

## Status

Accepted

## Date

2026-06-19

## Context

SIMPA handles aggregated health data for a municipal secretariat. Regressions in ETL parsing or API contract break trust with gestores. User selected full test coverage despite small/medium project size.

## Decision

Implement three test layers:

1. **Python unit tests** (`pytest`) — parser column detection, encoding ISO-8859-1, null semantics, consolidation output shape vs contract v3.1.0
2. **API integration tests** (`jest` + `supertest`) — auth flows, dashboard filters, import upload (mock Python subprocess), cadastros CRUD, contract JSON schema validation
3. **Playwright E2E** — login → Painel load → filter change → Importação upload (fixture CSV) → Cadastros navigation → logout; dark mode toggle smoke

CI runs all layers on push (GitHub Actions or local script). Docker Compose `test` profile spins ephemeral postgres for integration tests.

## Alternatives Considered

### Alternative 1: Pragmatic (Python unit + API smoke only)

- **Description**: Manual frontend QA checklist
- **Pros**: Less CI maintenance
- **Cons**: Regressions in multi-screen flows undetected
- **Why rejected**: User chose full strategy

### Alternative 2: API-only integration tests

- **Description**: No E2E
- **Pros**: Faster CI
- **Cons**: UI regressions (layouts A/B/C, auth redirect) uncaught
- **Why rejected**: Full redesign has complex navigation

## Consequences

### Positive

- Contract v3.1.0 guarded by schema tests
- E2E validates critical user journeys for Secretaria

### Negative

- Playwright adds CI time and flake risk
- Requires test fixtures (seed SQL, sample CSVs already in repo)

### Risks

- **Risk**: Flaky E2E on timing — **Mitigation**: `data-testid` attributes, wait for network idle on dashboard
- **Risk**: Subprocess mocking complexity — **Mitigation**: fixture JSON files for parser output in API tests

## Implementation Notes

- `tests/e2e/` in `simpa-frontend`
- `tests/` in `simpa-backend` and repo root for Python
- Embed test subtasks in each Compozy task (per cy-create-tasks skill)
